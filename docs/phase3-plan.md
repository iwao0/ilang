# ilang フェーズ3: 制御構造と真偽値

## Context

フェーズ2 までで `let` / `fn` / 型チェックまで揃ったので、実用的なロジックを書くために制御構造を入れる。Rust の所有権/`mut`/借用は採用せず、変数はすべて可変 (JS / Python 風) とする方針を確認した上での実装。

## 設計上の方針

- **変数は単一カテゴリ** (`let` のみ)。`mut` キーワードは無し
- **再代入は自由**: `let x = 1; x = 2;` は OK
- ARC により所有権/ライフタイム不要 (フェーズ未実装でも文法には影響しない)
- 制御フローは Rust スタイル (`if` は式、`{...}` ブロック)

## 実装した機能

### 真偽値型と比較
- `bool` 型と `true` / `false` リテラル
- 比較演算子: `==`, `!=`, `<`, `<=`, `>`, `>=`
- `==` `!=` は `bool` 同士でも可、順序比較は数値のみ
- 数値比較は `i64`/`f64` 混在可 (`1 == 1.0` → `true`)

### 論理演算 (短絡評価)
- `&&`, `||` (短絡)、`!` (前置)
- `false && (1 / 0 == 0)` → `false` (右辺は評価されない)

### 制御構造
- **`if` 式** (Rust 風): `let x = if a > 0 { 1 } else { -1 };`
- `else if` チェーン (`else` 直後の `if` を内部で `Expr::If` として保持)
- **`while` 文**: 条件は `bool` 必須、本体は `Unit` 必須 (値は捨てる前提なので明示的に `Unit` を要求)
- `if`/`while`/ブロックは「ブロック様式の式」として `;` 無しで文として並べられる

### 代入式
- `x = expr` は式 (型は `Unit`)、右結合 (`x = y = 1` → `x = (y = 1)`)
- 左辺は識別子のみ (将来フィールドアクセス対応時に拡張)
- 代入は **外側スコープに伝播**: `let x = 0; { x = 99; } x` → `99`
- ブロック内の `let` は **shadow**: `let x = 1; { let x = 99; } x` → `1`

## 演算子優先順位 (低 → 高)

| 演算子 | l_bp / r_bp | 結合性 |
| --- | --- | --- |
| `=` | 2 / 1 | 右 |
| `\|\|` | 3 / 4 | 左 |
| `&&` | 5 / 6 | 左 |
| `==` `!=` | 7 / 8 | 左 |
| `<` `<=` `>` `>=` | 9 / 10 | 左 |
| `+` `-` | 10 / 11 | 左 |
| `*` `/` `%` | 20 / 21 | 左 |
| 単項 `-` `+` `!` | — / 30 | 前置 |

`<` の l_bp=9 と `+` の r_bp=11 の組み合わせで `1 + 2 < 3` が `(1+2) < 3` になることを担保。

## 構文サンプル

```rust
fn count_div(n: i64): i64 {
    let i = 1;
    let count = 0;
    while i <= n {
        if i % 15 == 0 {
            count = count + 1;
        } else if i % 3 == 0 || i % 5 == 0 {
            count = count + 1;
        }
        i = i + 1;
    }
    count
}
count_div(100)   // => 47
```

## オープン課題 (フェーズ4 以降)

### 制御構造の拡張
- `loop` / `break` / `continue`
- `for` (反復子が入った後)
- `match` (ADT/`enum` とセット)
- `if let` / `while let`

### 言語コア
- 文字列型と文字列リテラル
- 配列 / `Vec`
- `struct` と `enum`
- ジェネリクス
- `Result` / `Option`

### サプライチェーン対策の本丸
- **capability の enforcement**: `#[requires(net)]` の関数を呼ぶ側にも `#[requires(net)]` 必須
- **モジュールシステム**: `use http_client with cap(net = env.net)` の形で import 時に明示的許可
- ライブラリ境界での capability 強制 (Wasm component model 参考)

### 実行時/コード生成
- ARC のランタイム実装
- `ilang-mir` 中間表現
- Cranelift 経由のネイティブコード化
